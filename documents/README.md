## 📝 ドキュメント更新ルール

### 更新時の注意事項
1. **日付の更新**: ドキュメント冒頭の「最終更新」日付を更新する
2. **変更履歴**: 大きな変更の場合は変更理由を記載
3. **関連ドキュメントの確認**: 影響するドキュメントも合わせて更新
4. **レビュー**: 重要なドキュメント変更はPRでレビューを受ける

### ドキュメント作成時の原則
- **具体的に**: 曖昧な表現を避ける
- **最新に保つ**: コードと同期を保つ
- **読みやすく**: 適切な見出し、箇条書き、図表を使う
- **検索しやすく**: キーワードを意識する

## 💡 よくある質問（FAQ）

### Q1: どのドキュメントから読めばいいですか？
A: まず **[プロジェクト概要](basic/プロジェクト概要.md)** と **[開発プロセスルール](../.cursor/rules/開発プロセスルール.md)** を読んでください。その後、役割別推奨ドキュメントを参照してください。

### Q2: ドキュメントが古い場合は？
A: 最新の情報はコードベース（`reserve-app/`）を確認し、ドキュメントをPRで更新してください。

### Q3: 新しいドキュメントを追加したい
A: 適切なフォルダに追加し、このREADME.mdも更新してください。

### Q4: AIエージェント（Cursor、Claude）向けの資料は？
A: **[開発プロセスルール](../.cursor/rules/開発プロセスルール.md)** がAIエージェント向けに最適化されています。
