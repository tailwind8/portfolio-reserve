# 事業展開戦略 - ポートフォリオ → 買い切り → SaaS

**最終更新**: 2026-01-01
**戦略**: 段階的にリスクを抑えながら事業拡大

---

## 🎯 全体戦略

```
Phase 1: ポートフォリオ完成（2週間）
   ↓ 就活・技術力アピール

Phase 2: 買い切り型で販売開始（3-6ヶ月）
   ↓ 実顧客獲得・ニーズ収集・即収益

Phase 3: SaaS化の判断（6ヶ月後〜）
   ↓ 共通ニーズが見えたら

Phase 4: SaaS事業化（1年後〜）
```

### この順番の理由

| フェーズ | メリット | リスク |
|---------|---------|--------|
| **ポートフォリオ** | 就活に使える、技術力証明 | なし |
| **買い切り** | 即収益、顧客理解、柔軟性 | 低（顧客がインフラ負担） |
| **SaaS** | 継続収益、スケール可能 | 中〜高（インフラ・保守負担） |

**ポイント**:
- ✅ 各フェーズで学習し、次の判断材料を得る
- ✅ リスクを最小化しながら段階的に拡大
- ✅ 途中でやめても損失が少ない

## 🚀 Phase 3: SaaS化の判断

**時期**: Phase 2開始から6ヶ月後
**目的**: SaaS化すべきか判断する

### SaaS化すべきサイン（5つ中3つ以上）

#### 1. 顧客数が増えた
- ✅ 買い切り顧客が10社以上
- ✅ 問い合わせが継続的にある
- ✅ 紹介による新規顧客が増えている

#### 2. 共通ニーズが明確
- ✅ カスタマイズ要望に共通パターンがある
- ✅ 「この機能は全顧客に欲しい」が3つ以上ある
- ✅ 個別対応していた機能を標準化できる

#### 3. 月額モデルへの需要
- ✅ 「初期費用が高い、月額で使いたい」という問い合わせ
- ✅ 保守契約を希望する顧客が多い
- ✅ 買い切り顧客から「月額に切り替えたい」の声

#### 4. 保守負担が増大
- ✅ 個別対応の工数が増えて大変
- ✅ 同じ機能を複数顧客に個別開発している
- ✅ バージョン管理が複雑になっている

#### 5. 収益の安定性を求めている
- ✅ 月額収益モデルで安定させたい
- ✅ 買い切りだけでは売上が不安定
- ✅ 継続収益を得たい

### SaaS化しない方が良いサイン

#### 1. カスタマイズが多様すぎる
- ❌ 顧客ごとに全く異なる要望
- ❌ 共通化できる機能が少ない
- ❌ 標準化すると誰も満足しない

#### 2. 買い切りで満足している
- ❌ 買い切り収益で十分満足
- ❌ インフラ管理の工数をかけたくない
- ❌ SaaSのサポート負担が重そう

#### 3. 顧客数が少ない
- ❌ 買い切り顧客が5社未満
- ❌ 新規問い合わせが月1件未満
- ❌ 市場ニーズが小さい

### 判断フローチャート

```
Q1: 買い切り顧客は10社以上？
  YES → Q2へ
  NO  → Phase 2継続（まだ早い）

Q2: 共通ニーズが3つ以上明確？
  YES → Q3へ
  NO  → Phase 2継続（ニーズ収集不足）

Q3: 「月額で使いたい」という問い合わせがある？
  YES → Q4へ
  NO  → Phase 2継続（需要不明）

Q4: 保守負担が増えて困っている？
  YES → SaaS化を検討開始
  NO  → 現状維持でもOK（焦らない）
```

## 💰 Phase 4: SaaS事業化

**時期**: Phase 3で「SaaS化すべき」と判断した後
**目的**: 継続収益モデルの構築

### SaaS化の準備

#### 技術面（3-6ヶ月）

1. **Row Level Security (RLS)** - データ分離強化
2. **サブドメインルーティング** - customer-a.yourdomain.com
3. **プラン制限機能** - スタッフ数・予約数の上限
4. **管理画面（運営者用）** - 全テナント管理
5. **課金システム** - Stripe統合
6. **監視・ログ** - エラー検知、パフォーマンス監視

#### ビジネス面（1-3ヶ月）

1. **価格設定**
   ```
   ライトプラン:     月額  5,000円（予約100件/月、スタッフ3人）
   スタンダード:     月額 10,000円（予約500件/月、スタッフ10人）
   プレミアム:       月額 20,000円（無制限 + カスタムドメイン）
   ```

2. **利用規約・プライバシーポリシー** - 弁護士監修推奨

3. **サポート体制**
   - メールサポート（営業日24時間以内返信）
   - チャットサポート（オプション）
   - FAQ充実

4. **マーケティング**
   - LP（ランディングページ）作成
   - SEO対策
   - 広告運用（Google Ads / Facebook Ads）

### SaaS版インフラ構成

```
┌─────────────────────────────────────┐
│         顧客A（customer-a）          │
│    customer-a.yourdomain.com        │
└─────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│       Vercel (Next.js App)          │
│   - 全顧客共通アプリケーション        │
│   - テナントIDで動的ルーティング      │
└─────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│   Supabase PostgreSQL（共有DB）      │
│   - tenant_id で論理的分離           │
│   - Row Level Security で保護       │
└─────────────────────────────────────┘
```

### SaaS版コスト試算（月額）

**10顧客の場合**:
```
【収益】
ライト 5社 × 5,000円   = 25,000円
スタンダード 3社 × 10,000円 = 30,000円
プレミアム 2社 × 20,000円 = 40,000円
─────────────────────────────
合計収益: 95,000円/月

【コスト】
Vercel Pro            3,000円
Supabase Pro          3,750円
Resend（メール送信）   1,500円
ドメイン              1,500円
─────────────────────────────
合計コスト: 9,750円/月

【粗利】
95,000 - 9,750 = 85,250円/月
利益率: 約90%
```
