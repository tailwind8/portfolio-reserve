## 📖 各ドキュメントの内容

### 1. README.md

**対象**: 全員
**所要時間**: 20-30分

**主な内容:**
- プロジェクト概要（システム名、コンセプト、主要機能）
- 技術スタック一覧
- プロジェクト構成（ディレクトリ構造）
- 開発の進捗状況（Sprint 1-4）
- セットアップ手順（環境構築）
- 開発プロセス（ATDD/BDD）
- テスト戦略（カバレッジ目標）
- CI/CD運用（GitHub Actions）
- データベース設計（Prismaスキーマ）
- 次のステップ（優先度別）
- トラブルシューティング
- 関連ドキュメント
- FAQ

**こんな時に読む:**
- プロジェクトに初めて参加する
- セットアップ方法を知りたい
- 開発プロセスを理解したい
- テスト実行方法を確認したい
- トラブルが発生した

---

### 2. 技術詳細.md

**対象**: 開発者（フロントエンド・バックエンド）
**所要時間**: 30-40分

**主な内容:**
- アーキテクチャ概要（システム構成図）
- 技術スタック詳細
- 認証システム（Supabase Auth）
- データフロー（ユーザー予約フロー、管理者フロー）
- API設計（RESTful API、エンドポイント一覧）
- コンポーネント設計（共通コンポーネント、設計原則）
- 状態管理（Server Components vs Client Components）
- バリデーション戦略（Zodスキーマ）
- エラーハンドリング（API、フロントエンド）
- パフォーマンス最適化
- セキュリティ対策

**こんな時に読む:**
- 新機能を実装する前に全体像を理解したい
- API設計を確認したい
- 認証システムの仕組みを知りたい
- コンポーネント設計の指針を確認したい
- セキュリティ対策を確認したい

---

### 3. 開発タスクリスト.md

**対象**: 開発者、プロジェクトマネージャー
**所要時間**: 15-20分

**主な内容:**
- 完了したタスク（Sprint 1）詳細
- 次のタスク（Sprint 2）詳細
  - Issue #8-14の具体的なタスクリスト
  - Gherkinシナリオ例
  - 受入基準
  - 関連ファイル
- 予定タスク（Sprint 3-4）概要
- 技術的負債・改善タスク
- 週次計画テンプレート
- 1日のルーチン
- PR作成前チェックリスト

**こんな時に読む:**
- 次に何をすべきか確認したい
- Sprint 2の作業内容を詳しく知りたい
- 週次計画を立てたい
- 技術的負債を確認したい
- 1日の開発フローを確認したい

## 📋 シチュエーション別ガイド

### 🆕 プロジェクトに初めて参加する場合

1. [README.md](./README.md) を読む（20-30分）
2. `.cursor/rules/開発プロセスルール.md` を読む（15分）
3. セットアップ手順に従って環境構築（30-60分）
4. サンプルテストを実行して動作確認（10分）
5. [技術詳細.md](./技術詳細.md) を読む（30-40分）
6. [開発タスクリスト.md](./開発タスクリスト.md) で次のタスクを確認（10分）

**合計所要時間**: 約2-3時間

---

### 🔧 新機能を実装する場合

1. [開発タスクリスト.md](./開発タスクリスト.md) で該当Issueを確認
2. `.cursor/rules/開発プロセスルール.md` で開発プロセスを確認
3. Gherkinシナリオを作成
4. [技術詳細.md](./技術詳細.md) でAPI設計・コンポーネント設計を確認
5. E2Eテスト作成
6. 実装
7. [README.md](./README.md) のPR作成前チェックリストを確認
8. PR作成

---

### 🐛 バグ修正する場合

1. [README.md](./README.md) のトラブルシューティングを確認
2. 該当する既存テストを確認
3. バグを再現するテストを追加
4. バグ修正
5. 全テスト実行
6. PR作成

---

### 📊 プロジェクト状況を把握する場合

1. [README.md](./README.md) の「開発の進捗状況」セクション
2. [開発タスクリスト.md](./開発タスクリスト.md) の「完了したタスク」
3. GitHub Issuesで未完了Issue確認
4. GitHub Actionsで最新のCI/CD状況確認

---

### 🚀 デプロイする場合

1. [README.md](./README.md) の「CI/CD運用」セクション
2. `documents/CI-CD運用ガイド.md`
3. `documents/Vercel環境変数設定.md`
4. GitHub Actionsワークフロー確認
